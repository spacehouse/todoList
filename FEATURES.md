# Features（v1.0.0）

TodoList 是一个运行在 Minecraft（Fabric）上的待办事项模组，支持单人游玩与多人服务器协作。本文聚焦描述 1.0.0 已具备的能力（用户向）。

## 任务管理

- 在游戏内 GUI 中创建、编辑、删除任务，并可随时标记完成/未完成
- 任务属性：标题、描述、优先级（低/中/高）、标签（逗号分隔）
- 搜索与筛选：按状态、优先级、文本（标题/描述）、标签实时过滤
- 可滚动任务列表：滚轮滚动 + 拖动滚动条；选中高亮与悬停反馈
- 已完成任务只读（不可编辑内容与优先级），仍可删除

## 项目与视图

- 支持个人项目与团队项目两种维度
- 侧边栏项目列表：搜索、切换项目；支持新建、编辑、删除项目
- 团队任务视图（多人服务器）：待分配、已分配、分配给我
- 团队项目支持成员管理与加入申请流程

## 多人协作与权限

- 多人下个人任务按玩家独立保存；团队任务由服务器共享存储并同步给玩家
- 团队视图的改动需要点击保存提交；点击取消或按 Esc 关闭会丢弃本地未保存改动并从服务器刷新，减少冲突
- 服务器侧权限中心按角色、视图范围、完成状态与指派关系统一校验团队操作
- 角色差异：
  - 管理员可完整管理团队任务（新增/编辑/删除/指派/完成）
  - 普通玩家可领取/放弃任务并完成分配给自己的任务；“指派他人”仅对管理员可见

## 指派与执行流

- 支持领取任务、放弃任务、指派他人，并提供“分配给我”视图聚焦当前玩家待办
- 指派人名称在离线场景可回退显示，保证 GUI/HUD 标签可读性

## HUD 展示与配置

- 右上角 HUD 显示当前视图的任务列表，支持展开/收起（H 键）与数量提示
- HUD 标题展示当前视图（个人 / 团队-待分配 / 团队-已分配 / 团队-分配给我）
- 真单人世界中 HUD 默认视图固定为个人视图且隐藏团队视图
- 内置 HUD 配置界面：宽度、最大高度、待办/已完成显示条数（0–30）、默认展开、空列表是否显示、默认视图、拖拽预览设置位置，并可保存并应用
- 若安装 Mod Menu，可从模组入口打开 HUD 配置界面

## 持久化与多语言

- 任务与项目数据以文件持久化保存（单人/多人均支持）
- 界面提供中文与英文语言包，可随游戏语言切换

## 审计日志

- 服务端会以统一 `[TEAM_OP]` 格式记录所有生效的团队操作，便于排查与审计
